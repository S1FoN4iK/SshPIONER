# SshPIONER

Бот работает в чате: кидаете ссылку на **TikTok**, **Instagram** (reel/пост/фото-карусель) или
**YouTube** → присылает медиа **без вотермарки и анонимно**.
Плюс следит за указанными TikTok-профилями и сам постит новые ролики, как только они выходят.

## Что нужно
- Python 3.11+
- [uv](https://docs.astral.sh/uv/)
- (опционально) `ffmpeg` — для основного пути не требуется; желателен для отдельных форматов yt-dlp

## Установка
```bash
uv sync
```

## Настройка Telegram
1. **@BotFather → `/newbot`** → получите `BOT_TOKEN`.
2. Если бот будет работать **в группе** — обязательно отключите privacy mode:
   **@BotFather → `/setprivacy` → выберите бота → Disable.**
   Без этого бот не видит обычные сообщения в группе и не поймает ссылки.
   После этого добавьте бота в группу.
3. Узнайте `TARGET_CHAT_ID` (куда поллер постит новые ролики): добавьте бота в нужный чат,
   перешлите сообщение из чата боту `@getidsbot`, либо добавить самого бота в ваш чат.
   Id группы будет отрицательным (супергруппы/каналы — с префиксом `-100`).

## Конфиг
Скопируйте `.env.example` → `.env` и заполните поля (минимум — `BOT_TOKEN`).
Для слежения за профилями нужны одновременно `TARGET_CHAT_ID` и `TIKTOK_PROFILES`.

### Прокси
При необходимости заполните `PROXY_URL` — и **весь** исходящий трафик (tikwm + yt-dlp) пойдёт через него.
Пусто → прямое соединение. `http(s)` и `socks5` работают из коробки
(`socks` благодаря extra `httpx[socks]`), креды можно зашить в URL:
`http://user:pass@host:port` или `socks5://user:pass@host:1080`.

### Cookies (для Instagram / возрастных видео)
Instagram почти всегда требует залогиненную сессию. Укажите одно из:
- `COOKIES_FILE` — путь к `cookies.txt` (формат Netscape), выгруженному из браузера;
- `COOKIES_FROM_BROWSER` — имя браузера (`chrome`, `firefox`, `edge`…), откуда брать cookies на лету.

## Запуск
```bash
uv run ttbot
# или
uv run python -m ttbot
```

## Как это работает / детали
- **TikTok** Сначала `tikwm.com` (поток без вотермарки в HD, плюс фото-посты),
  при сбое — фолбэк на `yt-dlp`. Порядок меняется в `PROVIDER_ORDER`.
- **Instagram.** Через `yt-dlp`: reels/видео, а также фото и карусели (несколько фото/видео
  уходят одним альбомом). Для большинства постов нужны cookies — см. выше.
- **YouTube.** Через `yt-dlp`. Разрешение ограничено `YOUTUBE_MAX_HEIGHT` (по умолчанию 720p),
  чтобы файл влезал в лимит Bot API без сборки `ffmpeg`.
- **Первый запуск слежки** за TikTok-профилем: бот запоминает текущие ролики и **не заливает
  весь бэклог** — постит только то, что выйдет после старта.
- **Лимит Telegram Bot API — 50 МБ.** Если файл больше `MAX_FILE_SIZE_MB`, бот пришлёт ссылку
  на оригинал вместо файла (для альбомов — пропустит слишком тяжёлые элементы и сообщит об этом).
  Снять лимит (до 2 ГБ) можно только подняв локальный Bot API server.

## Заметки
- Сторонние сервисы и сами платформы периодически меняют API — возможны временные поломки;
  для TikTok цепочка из двух провайдеров это сглаживает. Если что-то отвалилось —
  обновите `yt-dlp` (`uv lock --upgrade-package yt-dlp && uv sync`).

## Структура
```
src/ttbot/
  app.py       # сборка и запуск (long-polling + фоновый поллер)
  bot.py       # хэндлеры aiogram + отправка медиа (видео/фото/альбомы)
  poller.py    # фоновая проверка TikTok-профилей
  media.py     # модели, детект платформы, провайдеры (tikwm/yt-dlp), цепочка загрузки
  state.py     # хранилище просмотренных id
  config.py    # настройки из .env
```
