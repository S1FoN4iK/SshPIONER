# SshPIONER

Бот работает в чате: кидаете ссылку на TikTok → присылает видео **без вотермарки**.
Плюс следит за указанными профилями и сам постит новые ролики, как только они выходят.

## Что нужно
- Python 3.11+
- [uv](https://docs.astral.sh/uv/)
- (опционально) `ffmpeg` — для основного пути не требуется, желателен только для фолбэка yt-dlp

## Установка
```bash
uv sync
```

## Настройка Telegram
1. **@BotFather → `/newbot`** → получите `BOT_TOKEN`.
2. Если бот будет работать **в группе** — обязательно отключите privacy mode:
   **@BotFather → `/setprivacy` → выберите бота → Disable.**
   Без этого бот не видит обычные сообщения в группе и не поймает ссылки.
   После этого добавьте бота в группу.
3. Узнайте `TARGET_CHAT_ID` (куда поллер постит новые ролики): добавьте бота в нужный чат,
   перешлите сообщение из чата боту `@getidsbot`, либо добавить самого бота в ваш чат.
   Id группы будет отрицательным (супергруппы/каналы — с префиксом `-100`).

## Конфиг
Скопируйте `.env.example` → `.env` и заполните поля (минимум — `BOT_TOKEN`).
Для слежения за профилями нужны одновременно `TARGET_CHAT_ID` и `TIKTOK_PROFILES`.

## Запуск
```bash
uv run ttbot
# или
uv run python -m ttbot
```

## Как это работает / детали
- **Скачивание без вотермарки.** Сначала `tikwm.com` — он отдаёт поток без вотермарки в HD.
  Если не ответил/зарейтлимитил — фолбэк на `yt-dlp` (самодостаточный, без сторонних сервисов;
  у него отсутствие вотермарки — best-effort). Порядок меняется в `PROVIDER_ORDER`.
- **Первый запуск слежки** за профилем: бот просто запоминает текущие ролики и **не заливает
  весь бэклог** — постит только то, что выйдет после старта.
- **Лимит Telegram Bot API на загрузку — 50 МБ.** Ролики TikTok обычно меньше, но если видео
  больше `MAX_FILE_SIZE_MB`, бот пришлёт прямую ссылку без вотермарки вместо файла. Снять лимит
  (до 2 ГБ) можно только подняв локальный Bot API server.

## Заметки
- Сторонние сервисы и сам TikTok периодически меняют API — возможны временные поломки;
  цепочка из двух провайдеров это сглаживает. Если `tikwm` совсем отвалится — поставьте
  `PROVIDER_ORDER=ytdlp` и обновите `yt-dlp` (`uv lock --upgrade-package yt-dlp && uv sync`).

## Структура
```
src/ttbot/
  app.py       # сборка и запуск (long-polling + фоновый поллер)
  bot.py       # хэндлеры aiogram + отправка видео
  poller.py    # фоновая проверка профилей
  tiktok.py    # модели, провайдеры (tikwm/yt-dlp), цепочка загрузки
  state.py     # хранилище просмотренных id
  config.py    # настройки из .env
```
